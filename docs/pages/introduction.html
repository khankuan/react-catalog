What is React Catalog

React Catalog is a UI Component development tool, inspired by react-storybook and react-docgen, to help developers and designers build reusable UI Kit for their react application. As your application grows, the number of components will quickly grow out of hand. (As of July 2015, Facebook already has 15,000 components). This catalog is only concerned with Presentational components. The goal is to focus on pure UI design and development without worrying about the application logic (Redux, Relay, and more..). This documentation is also generated by the catalog (Checkout the /docs folder).

What does React Catalog provide

  • Organize Components - Categorize components by categories, making components searchable, browsable
  • Component Preview - Exploring a component by itself, great for development
  • Generated Component Docs - Interactive propTypes, linking dependencies
  • Custom Pages - Great for writing design guides, best practices for components, etc
  • Generated Snapshot Testing - Run test without writing test
  • Linter Integration - Hooked up with StandardJS
  • Transpile for exporting - Build your module into es5
  • Github Pages Integration - Single command to build and publish to Github Pages

What you need to do

  • Define propTypes - As detailed as possible, especially PropTypes.shape, isRequired
  • 1 component per file
  • Write comments - Categories, search tags, descriptions
  • Define stories - Various use cases and flows of a component
